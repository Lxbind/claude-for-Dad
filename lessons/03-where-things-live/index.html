Lesson 03

Where everything lives

~20 min visual folder map 4 quizzes cheat sheet

This is the lesson nobody teaches and everybody wishes they'd had. Before you install any Claude tools, you need to know where their stuff goes on your computer. Otherwise you'll have skills you can't find, projects you can't organize, and configs you can't edit.

Genuinely — the two-worlds rule (local vs. cloud) was the single thing that made everything else click for me when I was learning this. If it clicks for you, you're 60% of the way home. — L.

By the end you'll be able to answer: "I want to install a skill — where does it go? My settings? My project files? My claude.ai work?"

The two worlds

Claude work happens in two completely separate places, and confusing them is the #1 source of frustration:

  1. On your computer — where Claude Code, skills, and your actual work files live. Local files. You can see them in Finder/File Explorer.
  2. In the cloud — where claude.ai (the web app), claude.ai Projects, and your chat history live. You can't see these in your file explorer. They're on Anthropic's servers, attached to your account.

Mixing the two is the first mistake. The mental rule:

Files on disk = local Claude (Claude Code, Desktop, skills).
No files visible = cloud Claude (claude.ai, claude.ai Projects).

You uploaded a PDF to a "Project" inside claude.ai. Where does that file actually live?

  • In a folder on your computer
  • In the cloud, attached to your Anthropic account
  • In your browser's cache
  • In Claude Code's folder on your computer

Map 1 — On your computer

Here's the structure that works. Don't build this now — just see it.

Two key ideas in this tree:

~/.claude/ is Claude's home

Everything global to Claude Code lives in your home folder under .claude/. Note the leading dot — it's a hidden folder. On Mac, press Cmd+Shift+. in Finder to see hidden folders. On Windows, toggle "Hidden items" in File Explorer's View menu.

This is where:

  • Global skills go (~/.claude/skills/)
  • Your settings go (~/.claude/settings.json)
  • Global instructions go (~/.claude/CLAUDE.md)
  • Session history is auto-saved (~/.claude/projects/ — don't touch this)

Each project folder can have its own .claude/

A "project folder" in the Claude Code sense is just any folder where you keep work files. If you make a folder called ~/Documents/procurement/ and put your contracts, notes, and templates in it, that's your project.

Inside that folder, you can create a .claude/ directory with project-specific overrides:

  • A CLAUDE.md that tells Claude about your procurement role, your company's quirks, common acronyms (R&D, GxP, MSA)
  • A skills/ folder with skills that only apply when you're working in this folder

When you launch Claude Code inside ~/Documents/procurement/, it reads both ~/.claude/ (global) and ~/Documents/procurement/.claude/ (project). Project overrides win when they conflict.

You built a skill called weekly-supplier-summary that you'll use every week regardless of which folder you're in. Where do you put it?

  • ~/.claude/skills/weekly-supplier-summary/SKILL.md
  • ~/Documents/procurement/.claude/skills/weekly-supplier-summary/SKILL.md
  • In a claude.ai Project
  • ~/Documents/Skills/weekly-supplier-summary/SKILL.md

You built a skill called rfp-scorecard that's only useful when you're working in your procurement folder — it references your company's specific scoring rubric and shouldn't activate in other contexts. Where do you put it?

  • ~/.claude/skills/rfp-scorecard/SKILL.md (global)
  • ~/Documents/procurement/.claude/skills/rfp-scorecard/SKILL.md
  • ~/Documents/procurement/rfp-scorecard/SKILL.md (loose in the project)
  • As a claude.ai Project knowledge file

Map 2 — In the cloud (claude.ai)

This part has no folders you can see. It's all on Anthropic's servers, accessed via claude.ai.

Your Anthropic account (cloud)
│
├── Chats                       ← Every chat you've had, browsable in the sidebar
│
└── Projects                    ← claude.ai's "Projects" feature (different from Claude Code)
    │
    ├── "Procurement work"      ← A Project you create for ongoing procurement
    │   ├── Custom instructions ← "I'm a procurement consultant at..."
    │   ├── Knowledge files     ← PDFs / docs you upload (cloud-stored)
    │   └── Chats inside it     ← All inherit the instructions + knowledge
    │
    └── "Supplier research"     ← A different Project for ad-hoc supplier research
        └── (its own setup)

Three things to know:

  1. claude.ai Projects ≠ Claude Code project folders. They share the name and that's it. claude.ai Projects are cloud-only; Claude Code projects are folders on your disk.
  2. You can't open a claude.ai Project file in Finder. It only exists inside claude.ai. If you want to download it, you have to do it manually from the web UI.
  3. Custom instructions in a claude.ai Project work like a mini CLAUDE.md — they prefix every chat in that Project. Best for "I always need Claude to know I'm a procurement consultant at Sanofi."

You want every chat about procurement work to start with the same context paragraph ("I'm a procurement consultant at Sanofi, here's my role..."). You don't want to install Claude Code yet. Where does that paragraph go?

  • In a CLAUDE.md file on your computer
  • In a claude.ai Project's Custom instructions
  • Pasted at the start of every chat manually
  • In claude.ai's main settings

Where Claude Desktop fits

Claude Desktop is the macOS/Windows app version of claude.ai — same chat interface, but with MCP support (the "coworker" mode from Lesson 2).

Claude Desktop's stuff lives at:
  Mac:     ~/Library/Application Support/Claude/
  Windows: %AppData%\Claude\

Inside it:
  ├── claude_desktop_config.json   ← MCP server connections live here
  └── (other internal state — don't touch)

If you set up MCP for Gmail, Calendar, etc., that configuration lives in claude_desktop_config.json. You'll edit this file once during MCP setup. We're not setting it up now — just naming where it lives.

The cheat sheet

When in doubt, ask yourself: what kind of thing is this, and which world does it belong in?

You want to…It goes here
Install a skill you'll use everywhere~/.claude/skills/<name>/SKILL.md

Set global Claude Code instructions~/.claude/CLAUDE.md
Set procurement-only instructions~/Documents/procurement/CLAUDE.md
Keep contracts, notes, RFPs~/Documents/procurement/ (or wherever you want)

Configure MCP servers (Gmail, Calendar)claude_desktop_config.json (Mac: ~/Library/Application Support/Claude/, Windows: %AppData%\Claude\)

Print this cheat sheet or bookmark it. When you start installing things in Lesson 6+, you'll come back here.

Recommended setup for procurement

If you follow nothing else, do this:

  1. Create ~/Documents/procurement/ (or wherever, but a dedicated folder helps).
  2. When you install Claude Code (Lesson 6), it'll auto-create ~/.claude/ for you. Don't pre-create it.
  3. Inside ~/Documents/procurement/, create .claude/CLAUDE.md and put your role context there. That way Claude Code knows who you are whenever you're working in that folder.
  4. Keep your contracts, meeting notes, and RFPs inside ~/Documents/procurement/ so Claude Code can read them when you ask.
  5. Separately, in claude.ai, create a "Procurement" Project with the same role context in Custom instructions. Now both surfaces know who you are.

You don't have to do any of this today. The next lesson names the tools formally; the lesson after that has you match them to your work. Setup happens in Lesson 6.

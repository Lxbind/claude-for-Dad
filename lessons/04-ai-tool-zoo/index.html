Lesson 04

The AI tool zoo

~20 min 6 pick-the-answer questions one per tool

Claude isn't one product — it's a family of tools, each built for a different kind of work. This lesson names them. Lesson 5 maps them to your work.

Don't try to memorize these names. You'll forget them, then re-derive them in Lesson 5 against your own tasks. That re-derivation is what makes them stick. — L.

You've already seen two of them in action in Lesson 2 (Claude Code and Claude+MCP). This lesson names the rest and puts them in context.

Take a click-through tour of the four main surfaces first. Click each tab to see what it looks like — then the quizzes below will click.

Tool 1 — claude.ai (the web chat)

You open claude.ai in a browser. What's it best for?

  • Editing files on your computer
  • One-off thinking, drafting, summarizing pasted text — the all-purpose web chat
  • Sending emails on your behalf
  • Running code on your system

The mental model: claude.ai is the default surface. If you're not sure where to start a task, start here. You can always graduate to a more capable surface later.

Tool 2 — Claude Code

Claude Code runs in a terminal on your computer. What does it do that claude.ai can't?

  • It's a smarter version of Claude
  • It can read and edit files on your computer directly
  • It's the only one that can write code
  • It works offline

The mental model: Claude Code is for work on files. If your task is "read these 12 documents and synthesize them," Claude Code beats claude.ai because you don't have to paste each one in by hand.

For most procurement consultants, Claude Code is overkill at first. Don't install it until Lesson 4 reveals you actually need it.

Tool 3 — Skills

What is a Claude skill?

  • A way to make Claude smarter at a specific topic
  • A reusable instruction pack that auto-activates on a trigger phrase
  • A connection between Claude and an external app
  • A folder of files Claude can read from

You can stop here for the day — or continue to MCP, Projects, and Desktop ↓

The mental model: if you find yourself typing the same prompt structure for the third time, that's the trigger to make it a skill.

Tool 4 — MCP (Model Context Protocol)

What does MCP do?

  • It lets Claude read and write text
  • A standard for connecting Claude to external apps (Gmail, Calendar, your databases)
  • A faster version of Claude
  • A storage system for your past chats

The mental model: MCP is the answer to "can Claude check my calendar / send an email / search the company wiki?" Without MCP, no. With MCP, yes.

Setting up MCP requires some technical comfort. Don't worry about it for now — Lesson 4 will tell you whether you actually need it based on your workflow.

Tool 5 — Projects (in claude.ai)

In claude.ai, you can create a "Project." What does it give you that a normal chat doesn't?

  • Smarter responses
  • Persistent context — instructions and files that apply to every chat inside the project
  • Memory across all your chats
  • App connections, like email and calendar

The mental model: if you find yourself pasting the same setup paragraph into chat after chat, that setup belongs in a Project.

Tool 6 — Claude Desktop

Claude Desktop is the macOS/Windows app. When would you pick it over claude.ai (the web version)?

  • It's a smarter version of Claude
  • When you want Claude to use MCP connections (Gmail, Calendar, custom integrations)
  • It works offline
  • It's free, while claude.ai costs money

The mental model: Claude Desktop is claude.ai + MCP connections. If you don't need MCP yet, stay on claude.ai.

What a skill is, recap

A skill is a markdown file that tells Claude "when X happens, do Y." Claude loads it automatically when its trigger matches. Once installed, you never retype the prompt again.

The trigger: finding yourself running the same shape of prompt for the third time. If Lesson 6 had you draft a memo to a department lead, and you'll do that again next month, that's a skill candidate.

The four parts of a skill

  1. Name — short, lowercase, dashes-only. Becomes the folder name.
  2. Description — one sentence Claude reads to decide whether to activate this skill. Make it specific.
  3. When it activates — explicit trigger phrases or situations.
  4. What it does — the actual instructions Claude follows. Use the four-part recipe from Lesson 4 (role / task / context / constraints).

A worked example from Lesson 4 turned into a skill:

---
name: dept-lead-memo
description: Drafts a two-paragraph memo to an internal department lead
  about a procurement decision (pause, switch, escalate). Triggers when
  the user says "memo to [dept] lead" or pastes raw notes about a
  supplier decision.
---

# Department lead memo

## When this skill activates

When the user says "memo to [department] lead about [supplier]," or
pastes raw notes about a supplier decision and asks for a memo.

## What it does

Role: procurement consultant at Sanofi advising internal departments.

Task: Draft a two-paragraph memo to the named department lead
explaining the procurement decision. Tone: firm but collegial.

Context to extract from the user's notes:
- Which supplier
- What's changing (pause, switch, escalate)
- Why
- Who originally sponsored the supplier (if relevant)

Constraints:
- Two paragraphs, no more.
- Don't list missed deadlines as bullets — keep it narrative.
- Don't apologize for the decision.
- Don't promise a specific next step unless the user provided one.

Build yours

Fill in the form below. The right-hand pane shows what the resulting SKILL.md will look like. Click Copy or Download when you're done.

Install it

  1. Make sure Claude Code is installed on your computer (install guide).
  2. Create the folder (use your skill name from above):
    • Mac/Linux: ~/.claude/skills/<your-skill-name>/
    • Windows: C:\Users\<your-username>\.claude\skills\<your-skill-name>\
  3. Save the SKILL.md from above into that folder.
  4. Restart Claude Code (or open a new session).

Test it

Open Claude Code. Type the trigger phrase you defined under "When it activates." The skill should activate automatically — Claude will follow your instructions instead of giving a generic response.

If it doesn't activate, your description or trigger phrases probably need to be more specific. Iterate.

Done check

Did your skill activate when you triggered it?

  • Yes — Claude followed my instructions on the first try
  • It activated, but the output wasn't quite right
  • It didn't activate
  • I couldn't install it
